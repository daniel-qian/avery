```
Ingestion
负责“公司每天发生了什么”。
不做复杂判断，只负责接入、清洗、去重、存证。

Signals
负责“这些事情说明了什么变化”。
例如任务延期、协作减少、汇报和实际进度不一致。

Company RAG
负责“这家公司自己的背景是什么”。
包括公司资料、项目文档、历史决策、团队结构。

Capabilities RAG
负责“遇到这种情况，一般专业上怎么处理”。
来自专家 SOP、真实业务案例、行业 playbook。

Evidence Builder
负责“这次回答到底该看哪些证据”。
避免把所有信息都塞给模型，也避免模型凭空发挥。

Agent Orchestration
负责“该让哪个 agent、哪个模型、哪种安全策略处理”。
普通问题快答，高风险问题走更严格流程。

Execution
负责“把建议变成实际动作”。
任务、owner、deadline、follow-up、artifact。

Review
负责“关键判断由人确认”。
manager 可以接受、修改、驳回，系统把反馈纳入下一轮判断。
```
